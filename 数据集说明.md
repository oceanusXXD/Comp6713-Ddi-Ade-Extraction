#  COMP6713 医疗关系抽取数据集说明文档 (ADE & DDI)

本仓库提供的数据集已完成标准化处理，旨在支持基于 **Qwen3.5-4B** 的多任务信息抽取（Information Extraction）任务 。

## 1. 数据存储格式 (Data Structure)

数据集采用 **ChatML (JSONL)** 格式。每一行都是一个完整的对话对象，包含三个核心角色（Roles），模型在微调时将学习从 `user` 文本中提取关系并生成 `assistant` 的 JSON 结果。

### 格式示例：
```json
{
  "messages": [
    {
      "role": "system", 
      "content": "You are an expert medical information extraction system... (包含了任务指令与标签定义)"
    },
    {
      "role": "user", 
      "content": "Erythromycin may increase the serum concentration of simvastatin."
    },
    {
      "role": "assistant", 
      "content": "[{\"head_entity\": \"Erythromycin\", \"tail_entity\": \"simvastatin\", \"relation_type\": \"DDI-mechanism\"}]"
    }
  ]
}
```

* **`system`**: 注入任务约束和 5 种标签的英文详细定义。
* **`user`**: 原始医疗文本（来自医学论文摘要或临床记录）。
* **`assistant`**: **严格的 JSON 列表字符串**。即使没有提取到关系，也必须输出 `[]` 。

---

## 2. 抽取内容与标签定义 (Label Definitions)

模型被要求识别并分类以下 5 种特定的医疗关系：

| 关系标签 | 所属任务 | 医学含义说明 |
| :--- | :--- | :--- |
| **`ADE`** | 不良反应抽取 | **药物导致了负面生理后果**。例如：药物 A 引起了皮疹。 |
| **`DDI-mechanism`** | 药物相互作用 | **药代动力学互动**。描述药物在体内如何影响另一药物的代谢、吸收或血药浓度。 |
| **`DDI-effect`** | 药物相互作用 | **药效学互动**。描述合用后产生的具体临床后果，如“毒性增加”或“药效增强”。 |
| **`DDI-advice`** | 药物相互作用 | **临床用药建议**。直接给出的警告或指令，如“不建议同服”或“需调整剂量”。 |
| **`DDI-int`** | 药物相互作用 | **未指明的互动**。仅提到存在相互作用，但未说明具体机制或后果。 |

---

## 3. 数据划分逻辑 (Data Splits)

为了保证多任务学习的稳定性，我们对 ADE 和 DDI 的原始数据进行了交叉合并与打乱：

1.  **`mixed_chatml_train.jsonl`**: 核心训练数据，包含了 ADE 训练集与 90% 的 DDI 训练集。
2.  **`mixed_chatml_val.jsonl`**: 混合验证集，使用 ADE 官方验证集并补入了 10% 的 DDI 训练数据，用于同步监控两类任务的收敛情况。
3.  **`mixed_chatml_test.jsonl`**: 最终测试集，合并了两者的测试数据，作为评估 Precision, Recall 和 F1-score 的唯一依据。

---

## 4. 关键注意事项 (Data Integrity)

* **JSON 完整性**: `assistant` 输出的内容必须通过 `json.loads()` 解析，字段名固定为 `head_entity`, `tail_entity`, `relation_type` 。
* **全局洗牌 (Shuffle)**: 训练集已经过随机打乱，请勿按顺序读取，以防模型产生任务偏见。
* **英文指令**: 系统提示词（System Prompt）采用全英文，以匹配 Qwen 模型底层的医学语义空间。
