# 同风格核心评测集说明

这个目录不是一个新的公开数据集，它只是把仓库里现有的“同风格 held-out 文件”整理成一个统一的评测入口。

## 当前文件说明

- `README.md`
  这个说明文件。
- `official_held_out/merged_chatml_validation.jsonl`
  同风格 held-out 验证集镜像。
- `official_held_out/merged_chatml_test.jsonl`
  同风格 held-out 测试集镜像。

## 它实际对应仓库里的什么文件

这两个文件当前对应的是：

- `../../data/merged_chatml_validation.jsonl`
- `../../data/merged_chatml_test.jsonl`

也就是说，这里只是额外放了一份评测入口，不是单独重新构建了一套新数据。

## 它到底是“用了”还是“没用”

结论直接写在这里：

- 主线训练：没用
- 默认验证：没直接用
- 默认测试：没直接用
- 额外同风格评测：可以用

所以如果你问“这个数据集当前主线是不是在用”，答案是：

- 主线配置没有直接用它
- 但你完全可以把它当成一个补充评测包来手动使用

## 为什么要保留这个目录

因为 `evaluate_datasets/` 目录的目标是把所有评测入口放到一起管理：

- 外部公开迁移评测包放这里
- 同风格 held-out 镜像也放这里

这样后续无论做内部评测还是外部迁移评测，都能从同一层目录开始查。

## 当前没有包含什么

当前没有把下面这些内部子集单独物化出来：

- `hard`
- `negative`
- `schema_stability`

原因是仓库里目前还没有给出可复现的切分规则，也没有足够元数据支撑无歧义重建。
