# 报告目录说明

这个目录保存数据审计、训练摘要和统计结果，是理解“当前主线怎么整理出来的”的最好入口之一。

## 当前文件

- `data_audit_report.md`
  数据审计报告，记录清洗、污染排查和处理说明。
- `data_fix_log.md`
  数据修复日志，记录修过什么、为什么修。
- `data_stats.json`
  数据统计结果的结构化 JSON。
- `train_command.txt`
  训练命令记录，便于复现。
- `train_summary.md`
  训练结果摘要，包含主线数据、参数和运行状态概览。

## 什么时候看这里

- 想确认数据做过哪些修订
- 想知道主线数据是怎么从原始版本整理出来的
- 想复现上一轮训练命令
- 想先看摘要再回头读配置文件

## 使用建议

- 看数据来龙去脉：先读 `data_audit_report.md`
- 看统计规模：再看 `data_stats.json`
- 看当前训练概况：最后看 `train_summary.md`
